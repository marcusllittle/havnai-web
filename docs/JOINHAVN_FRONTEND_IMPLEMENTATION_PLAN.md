# JoinHavn Front-End Modernization Implementation Plan

_Last updated: 2026-04-09_

## Objective
Implement a real flagship-quality redesign of JoinHavn so the site feels premium, futuristic, cinematic, and clearly structured, with Astra as the emotional front door and HavnAI as the engine underneath.

---

## Phase 1: Foundation / Design System

### 1. Establish the new visual system
Build a reusable visual language before touching page-level composition.

#### Tasks
- Define new color tokens for:
  - deep background layers
  - glass/panel surfaces
  - glow accents
  - muted text / primary text / CTA emphasis
- Define spacing scale for premium large-format layout
- Define typography hierarchy:
  - display headline
  - section headline
  - body copy
  - eyebrow / metadata labels
- Define motion rules:
  - reveal on scroll
  - hover depth
  - section transitions
  - atmospheric background motion
- Define reusable premium components:
  - hero CTA buttons
  - spotlight cards
  - glass panels
  - section headers
  - stat chips
  - ecosystem flow blocks

#### Deliverable
A reusable front-end style system that can support the entire redesign.

---

## Phase 2: Homepage Redesign

### 2. Rebuild the homepage as a narrative flagship landing page
The homepage is the highest-priority surface and should set the tone for the entire ecosystem.

#### Section order
1. Cinematic Hero
2. Ecosystem Identity
3. Astra Spotlight
4. HavnAI Engine
5. Core Loop (Create → Own → Earn → Play)
6. Product Surface Grid
7. Operator / Network Layer
8. Final CTA

#### Tasks
- Replace current alpha-dashboard hero structure
- Build a premium Astra-led hero with cinematic visuals
- Add a simple visual explanation of JoinHavn / Astra / HavnAI relationship
- Add Astra showcase block with premium media treatment
- Add “engine underneath” HavnAI section
- Add clear ecosystem loop section
- Add premium destination cards for key surfaces
- Move operator / node onboarding lower on page
- End with multi-role CTA band

#### Deliverable
A fully redesigned homepage that feels cinematic, premium, and clear.

---

## Phase 3: Navigation + Information Architecture

### 3. Modernize nav and top-level wayfinding
The site structure must reflect the actual product hierarchy.

#### Tasks
- Redesign SiteHeader for flagship feel
- Replace vague “Join” label with “Run a Node” or “Operator”
- Add clearer route grouping:
  - Astra
  - Create
  - Library
  - Marketplace
  - Credits
  - Network
  - Run a Node
- Review mobile nav treatment to ensure premium, not cramped
- Ensure nav prioritizes emotional/product destinations first, infra second

#### Deliverable
Top-level navigation that supports the new ecosystem story.

---

## Phase 4: Key Surface Modernization

### 4. Upgrade high-impact pages to match the new flagship system
Once homepage is modernized, the supporting pages need to visually match.

#### Priority order
1. Generator
2. Library
3. Marketplace
4. Credits / Pricing
5. Run a Node
6. Analytics / Network pages

#### Tasks by surface

##### Generator
- Make it feel premium and creator-first
- Improve hierarchy between prompting, model selection, and output
- Reduce “tool panel” feeling where possible
- Make result states feel impressive and ownable

##### Library
- Reframe as ownership hub, not local job list
- Improve collection presentation
- Surface actions clearly: view, download, list, use
- Stronger premium asset-grid feel

##### Marketplace
- Make it feel like a real premium exchange, not a utility page
- Improve featured listings / browse hierarchy
- Clarify buy / own / relist flow

##### Credits / Pricing
- Explain Credits vs HAI clearly
- Simplify purchase hierarchy
- Remove unnecessary ambiguity around on-chain vs off-chain value

##### Run a Node
- Reposition as operator onboarding
- Keep technical clarity, but improve presentation quality
- Make node participation feel prestigious and ecosystem-critical

##### Analytics / Network
- Preserve data utility, but modernize presentation
- Reduce internal admin vibe
- Emphasize network vitality and credibility

#### Deliverable
Supporting pages that visually and structurally align with the flagship homepage.

---

## Phase 5: Astra Integration as Emotional Layer

### 5. Make Astra feel intentional throughout the site
Astra should not feel like a side-link. It should be woven into the ecosystem.

#### Tasks
- Add Astra showcase treatment to homepage
- Add Astra references in ecosystem loop
- Add “use in Astra” or “connected to Astra” moments where relevant
- Position Astra as the world where outputs, rewards, and identity matter emotionally
- Ensure Astra-related CTAs are visually premium and not buried

#### Deliverable
Astra integrated into the site as the emotional centerpiece, not an afterthought.

---

## Phase 6: Economy Clarity

### 6. Make the economic model understandable without crypto overload
The site needs to modernize while becoming easier to understand.

#### Tasks
- Create a simple Credits vs HAI explanation module
- Reuse that explanation consistently across surfaces
- Ensure “credits = spend” and “HAI = earn/own” is always legible
- Make wallet interaction feel supportive, not mandatory too early
- Avoid leading with tokenomics in hero areas

#### Deliverable
A clean economic explanation layer that supports confidence instead of confusion.

---

## Phase 7: Motion / Atmosphere / Polish

### 7. Add premium motion and cinematic atmosphere
Once the core layouts are rebuilt, add the finishing layer.

#### Tasks
- Ambient background motion
- Section reveal choreography
- Subtle parallax and depth effects
- Hover states for spotlight cards and CTAs
- Media treatment for Astra and showcase surfaces
- Soft luminous transitions between dark sections

#### Deliverable
A site that feels alive and flagship-level without sacrificing usability.
